# @vue/cli卸载不了？

大家好，我是前端小贺，希望能够通过自己的学习输出给你带来帮助。

# 背景

为了尝试用yarn安装@vue/cli，准备卸载掉之前用npm安装的@vue/cli。结果用vue -V验证是否卸载成功时，竟然还提示我了对应的版本号@vue/cli 4.5.3，没卸载成功！以下是我的操作步骤：

➜  ~ npm list -g --depth 0
/Users/hsm/.nvm/versions/node/v16.13.1/lib
├── @vue/cli@4.5.3
├── corepack@0.10.0
├── npm@8.1.2
├── pnpm@6.32.3
└── yarn@1.22.18

➜  ~ vue -V
@vue/cli 4.5.3
➜  ~ npm uninstall -g @vue/cli

removed 881 packages, and audited 1 package in 2s

found 0 vulnerabilities

# 尝试

  1. 考虑到我安装了nvm来管理NodeJs， 所以我查找了我所有下载过的版本并逐一排查了每个版本的npm全局安装包中是否有@vue/cli，有的话就uninstall。最终删除完了所有Node版本的@vue/cli后，我在命令行工具使用最常用的V16.13.1版本中输入vue -V竟然还显示@vue/cli 4.5.3! What the fuck?

  1. stackoverflow上找资料，然后尝试：

    1. 使用which vue命名来查看vue可执行文件的路径

      ➜  ~ which vue
/usr/local/bin/vue
      1
      2

    2. 删除文件，然后查看是否删除了

      ➜  ~ rm /usr/local/bin/vue
➜  ~ vue -V
zsh: command not found: vue
      1
      2
      3

    Ok，大功告成！
